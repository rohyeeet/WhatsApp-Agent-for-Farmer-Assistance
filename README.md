## 🎯 Overview

Kisan Mitra is an advanced multilingual agricultural assistant built using Google's Agent Development Kit (ADK). It serves as a comprehensive digital companion for Indian farmers, providing expert guidance across all aspects of modern agriculture, delivered directly inside **WhatsApp** — the channel farmers already use every day.

## 🌍 Business Context

**Mission**: *"किसान हमारे अन्नदाता हैं - हमारा कर्तव्य है उनकी सेवा करना!"* — "Farmers are our food providers, it is our duty to serve them."

**Problem**: Small and marginal Indian farmers rarely have easy, personalized access to the information that drives good farming decisions — hyperlocal weather, what to sow this month, today's mandi (market) price, which government subsidy they qualify for, or what disease is attacking their crop. Much of this information exists online but is scattered, in English, and not built for a farmer holding a basic smartphone.

**Solution**: Kisan Mitra puts a conversational agricultural advisor directly inside WhatsApp. A farmer can text or send a voice note in Hindi/Hinglish/English, or photograph a diseased crop leaf, and get a short, actionable, personalized reply in their own language and script, generated by a Gemini-powered agent that remembers who they are, where they farm, and what they grow.

**Target users**: Indian farmers, especially small/marginal landholders, who are more comfortable with voice and regional language than typing English into a web app.

**Core value propositions**:
1. **Zero learning curve** — it's just a WhatsApp chat.
2. **Personalization** — the agent remembers the farmer's name, village, land size, crops, and past conversations, and tailors every answer.
3. **Multilingual & voice-first** — replies mirror whatever language/script the farmer used, and voice notes are supported both in and out.
4. **Authoritative, current data** — weather, mandi prices, and scheme details instead of generic advice.
5. **Trust & tone** — the agent behaves like a knowledgeable friend, not a corporate bot; short WhatsApp-length replies, not essays.

## 🤖 The Agent

Defined in `kisan_mitra/agent.py`:

```python
root_agent = Agent(
    name='kisan_mitra',
    model='gemini-2.5-flash',
    description='Kisan Mitra (किसान मित्र) - Your personal agricultural advisor...',
    instruction=KISAN_MITRA_PROMPT,
    tools=[ ...22 tool functions... ],
)
```

- **Model**: `gemini-2.5-flash` — chosen for low latency (WhatsApp-scale conversational turnaround) and native support for text + image input (used for crop-disease photos).
- **System prompt (`KISAN_MITRA_PROMPT`)** encodes the product's behavioral rules:
  - **Language mirroring** — always reply in the exact language/script (Hindi, English, or Hinglish) the farmer used.
  - **Tone** — a conversational friend, not a corporate assistant; concise, WhatsApp-length responses (well under ~300 words).
  - **Domain scope** — crop lifecycle (sowing/growth/pests/fertilizer/harvest), weather, government schemes, mandi prices, livestock, and sustainable/carbon farming.
  - **Tool discipline** — load the farmer profile only when personalization is needed; check weather before giving crop advice; avoid unnecessary tool calls that add latency.
  - **Memory strategy** — after informative exchanges, call `save_farmer_insight` with a compact (<150 char) actionable note; for returning farmers, call `get_farmer_context_summary` first to resume context.
  - **Voice replies** — kept conversational, not essay-formatted, since they'll be spoken back via TTS.
  - **Proactive recognition** — capture and store name, village, crops, and land size whenever the farmer mentions them, and address the farmer by name.

## 🛠️ Tools Catalogue (22 registered ADK tools)

All tools live under `tools/` and are re-exported through `tools/__init__.py`. Per the project's ADK convention (see [CONTRIBUTING.md](CONTRIBUTING.md)), tool functions take **no default parameters**, return `Dict[str, Any]`, and surface errors as Hindi-language messages so the agent can relay them directly.

### 1. Farmer Profile & Memory — `tools/memory_tools.py`, `tools/farmer_context_tools.py`
Persistent storage of who the farmer is and what the agent has learned about them.

| Function | Purpose |
|---|---|
| `load_farmer_profile(user_id)` | Full profile + last 15 saved insights |
| `get_farmer_context_summary(user_id)` | Compact profile snapshot (name, location, language, crops, challenges) for quick reuse each turn |
| `get_crop_specific_context(crop_name, user_id)` | Farmer's own data for one crop (variety, area, growth stage) |
| `get_seasonal_recommendations(user_id)` | Priority actions for the current Kharif/Rabi/Zaid season |
| `update_farmer_profile_field(user_id, category, field, value)` | Generic nested-field update |
| `update_farmer_location(user_id, district, state, village)` | Sets location (also drives weather/mandi lookups) |
| `update_farmer_name(user_id, name, name_english)` | Records the farmer's name |
| `add_farmer_crop(user_id, crop_name, season)` | Adds a crop to the farmer's cropping pattern |
| `save_farmer_insight(user_id, insight)` | Appends a short "what we learned" note (rolling last 50) |

**Storage**: Google Cloud Firestore, collections `farmers/{user_id}` and `sessions/{user_id}`. If Firestore credentials aren't configured, the module transparently falls back to in-process dictionaries so the agent keeps working in a degraded (non-persistent) mode during development.

**Default profile schema** (created on first contact) captures: personal info, location + agro-climatic zone, farm size/soil/water sources, Kharif/Rabi/Zaid cropping pattern, challenges faced, communication preferences (language/timing/format), economic profile (input budget, insurance, market linkages), input usage, and enrolled government schemes.

### 2. Weather Intelligence — `tools/weather_tool.py`
| Function | Purpose |
|---|---|
| `get_farmer_weather(user_id)` | Resolves the farmer's stored district/state, then fetches weather |
| `get_agricultural_weather(location, language)` | Weather for an arbitrary location string |

- **External API**: OpenWeatherMap — Geocoding → Current Weather → 3-day Forecast.
- Derives **agricultural insights**, not just raw numbers: spray suitability (blocked by high wind or high humidity), irrigation advice, field-work suitability, and disease-risk flags (e.g. humidity > 80% → fungal disease risk; temp > 35°C → advise against midday work; rain/thunderstorm → postpone spraying).

### 3. Farming Calendar — `tools/farming_calendar_tool.py`
| Function | Purpose |
|---|---|
| `get_farming_calendar_advice(location, month)` | Month-by-month seasonal guidance (defaults to current month) |
| `get_crop_specific_calendar(crop_name)` | Full lifecycle detail for one crop |

- Backed by `context/farming_calendar_dataset.json`: a 12-month calendar (main crops, vegetable crops, sowing prep, pest/disease watch, weather considerations, regional variation across North/South/East India, market trends) plus a `crop_database` with ~40+ crops (scientific/Hindi/regional names, sowing & harvest months, soil/water needs, major varieties, fertilizer recommendations).

### 4. Government Schemes — `tools/agriculture_schemes_tool.py`
| Function | Purpose |
|---|---|
| `get_relevant_schemes_for_farmer(user_id)` | Personalized shortlist based on the farmer's profile |
| `get_scheme_details(scheme_name, user_id)` | Full details for one scheme |
| `list_all_available_schemes(user_id)` | Full database listing |
| `search_government_schemes(query)` | Live Google search fallback for schemes not yet in the local dataset |

- Backed by `context/agriculture_schemes.json` — ~15 schemes including PM-KISAN, PMFBY (crop insurance), Kisan Credit Card, Soil Health Card, PMKSY (irrigation subsidy), eNAM, PKVY (organic farming), Agriculture Infrastructure Fund, and the National Beekeeping & Honey Mission. Each entry includes eligibility, benefits, application process, required documents, and the official ministry/website.

### 5. Market (Mandi) Intelligence — `tools/mandi_prices_tool.py`
| Function | Purpose |
|---|---|
| `get_farmer_mandi_prices(user_id)` | Today's prices near the farmer |
| `get_mandi_prices_for_date(date, user_id)` | Historical price lookup |
| `get_commodity_price_info(commodity, user_id)` | Price for one commodity |

- **Fallback chain**: live API attempt → web-scrape attempt → **regional/seasonal estimate model** (state-level price multipliers × month-based seasonal adjustment factors) covering 10 major commodities (Wheat, Rice, Potato, Onion, Tomato, Cotton, Sugarcane, Mustard, Gram, Soyabean). Every estimated response is explicitly labeled as an estimate with a disclaimer to confirm with the local mandi.

### 6. Voice Processing — `tools/voice_processing_tool.py`, `tools/simple_voice_transcription.py`
| Function | Purpose |
|---|---|
| `process_voice_input()` | Speech-to-text |
| `generate_voice_response(text)` | Text-to-speech |
| `check_voice_service_status()` | Reports which STT/TTS backends are currently available |

- **Speech-to-text priority chain**: ElevenLabs Scribe v1 → Google Cloud Speech → Gemini multimodal transcription (retry w/ backoff). Supports 13 Indian languages (Hindi, English, Punjabi, Bengali, Tamil, Telugu, Marathi, Gujarati, Kannada, Malayalam, Odia, Assamese, Urdu).
- **Text-to-speech priority chain**: ElevenLabs Multilingual v2 (voice "Sarah") → gTTS as a free fallback.
- Designed so voice features **degrade gracefully to text-only** if no voice API keys are configured — the rest of the agent keeps working.

### 7. Disease Detection
There is no separate "vision tool" — crop-disease diagnosis is handled by Gemini's **native multimodal input**. When a farmer sends a photo over WhatsApp, the gateway base64-encodes the image and attaches it as `inlineData` alongside the text message in the same ADK `/run` call; the model reasons over the image directly as part of its normal response.

## 📲 WhatsApp Gateway

`whatsapp_kisan_mitra.py` is a FastAPI service (port 8000) and the only public-facing component; the ADK agent server is intentionally kept internal.

**Endpoints**
- `GET /` and `GET /health` — health check.
- `POST /webhook/whatsapp` — the Twilio webhook. Twilio requires a response within ~2 seconds, so the handler immediately returns empty TwiML and does all real work in a background task.
- `POST /test/message` — bypass Twilio entirely for local testing (`message`, `phone`, optional `voice` flag).
- `GET /test/voice-status` — surfaces STT/TTS backend availability.
- `/staticv2/*` — static file serving for generated voice-reply MP3s so Twilio can fetch them by URL.

**Message flow (`_process_and_reply`)**
1. **Voice detection** — if the incoming media's content-type contains `audio`/`voice`/`ogg`, treat it as a voice note.
2. **Transcription** — download the audio from Twilio (with Twilio basic auth) and transcribe it using the farmer's known preferred language.
3. **Image handling** — for image attachments, download and base64-encode the bytes for multimodal input.
4. **Session management** — one ADK session per farmer (`userId` = phone number). Sessions are created via the ADK server's session API, cached in Firestore (`set_active_session`/`get_active_session`), and expire after 1 hour of inactivity; a 404 from the ADK server triggers session recreation and a single retry.
5. **Agent invocation** — `POST http://localhost:8001/run` with the ADK-standard payload (`appName`, `userId`, `sessionId`, `newMessage.parts`), wrapped in `tenacity` retry logic (up to 5 attempts, exponential backoff) that specifically retries on HTTP 429/5xx.
6. **Voice reply generation** — if the inbound message was voice, the outbound text reply is also converted to speech and saved under `staticv2/` as a timestamped MP3, served over the public tunnel/Cloud Run URL.
7. **Reply delivery** — long replies (>1500 chars) are split at sentence/newline boundaries into multiple WhatsApp messages; the audio attachment (if any) rides on the first chunk. If Twilio rejects a media URL, the code retries as a text-only message.
8. **Insight persistence** — after every non-trivial exchange, a compact "Q / A" note is saved to the farmer's profile via `save_farmer_insight` so future turns (and future sessions) have continuity.

**Local dev networking**: the gateway can auto-detect a running ngrok tunnel (via its local API on port 4040) to build a public callback URL for Twilio media.

## ⚙️ Process Topology

- `main.py` simply re-exports `root_agent` so the ADK CLI/tooling can discover it.
- `runner.py` is the **service manager**: it launches `adk api_server --host 0.0.0.0 --port 8001` and `python whatsapp_kisan_mitra.py` as sibling subprocesses, pipes both logs to stdout with a prefix, and monitors both — if either process dies, the manager exits non-zero so the container orchestrator can restart it.
- `entrypoint.sh` exists because **Cloud Run's filesystem is read-only** outside `/tmp`. It copies the whole app into `/tmp/adk_home`, `cd`s there, and `exec`s `runner.py` — giving the runtime a writable working directory for logs, Firestore's local credential probing, and generated voice files.

## 🚢 Deployment

- **Containerization** (`Dockerfile`): `python:3.10-slim` base, with `build-essential` (compiling native Python deps) and `ffmpeg` (audio transcoding for voice) installed, `pip install -r requirements.txt`, then `entrypoint.sh` as the container command. Exposes port 8000.
- **Target platform** (`deploy_gcp.sh`): Google Cloud Run — a fully managed, serverless container platform. The script verifies `gcloud` is installed, resolves/sets the GCP project, enables the Cloud Run and Container Registry APIs, and deploys straight from source (`gcloud run deploy kisan-mitra-bot --source . --region us-central1 --allow-unauthenticated --port 8000`), forwarding the local `.env` file's contents as Cloud Run environment variables. `--allow-unauthenticated` is required because Twilio must be able to reach the public webhook without Google auth.
- **Local development**: `start_kisan_mitra_whatsapp.sh` starts the ADK server, waits for it to be healthy, starts the WhatsApp gateway, waits for its `/health`, starts `ngrok http 8000` for a public tunnel, extracts the ngrok URL from its local API, and prints the exact Twilio webhook URL to configure — plus ready-to-run `curl` smoke tests. `stop_system.sh` tears everything down by PID files and `pkill` fallback. `monitor_logs.sh` tails/multitails the running logs.

## 🚀 Quick Start

### Prerequisites
- Python 3.8 or higher
- Google API Key (for ADK)
- OpenWeather API Key (optional, for weather features)
- Twilio Account (for WhatsApp gateway)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/rohyeeet/WhatsApp-Agent-for-Farmer-Assistance.git
   cd WhatsApp-Agent-for-Farmer-Assistance
   ```

2. **Create virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure environment**
   ```bash
   cp .env.example .env
   # Edit .env with your API keys — see Configuration below
   ```

5. **Run the agent (ADK web UI only)**
   ```bash
   adk web --no-reload
   ```

   **Or run the full WhatsApp stack (ADK server + WhatsApp gateway)**
   ```bash
   ./start_kisan_mitra_whatsapp.sh
   ```

- `test_responses.py` — end-to-end smoke test: opens ADK sessions and sends four representative prompts (English greeting, Hindi sowing question, weather query, scheme query) to validate the full agent pipeline.
- `test_twiml.py` — validates TwiML XML generation/escaping for text + audio replies.
- `test_app.py` — a minimal standalone ADK agent used to sanity-check the ADK installation itself, independent of the full Kisan Mitra tool set.

## 📊 Capability Summary

| Capability | How it's delivered |
|---|---|
| Natural-language, multilingual chat | `gemini-2.5-flash` + language-mirroring system prompt |
| Voice-in / voice-out conversations | ElevenLabs/Google Cloud Speech/Gemini STT, ElevenLabs/gTTS TTS |
| Crop disease diagnosis from photos | Native Gemini multimodal image reasoning |
| Personalized, memory-driven advice | Firestore-backed farmer profile + rolling insight log |
| Hyperlocal weather & agronomic guidance | OpenWeatherMap + rule-based agricultural interpretation |
| Seasonal crop calendar | Curated 12-month, 40+ crop JSON dataset |
| Government scheme discovery | Curated dataset + live search fallback |
| Mandi price intelligence | Estimate model with regional/seasonal adjustment, date-aware |
| Reliable delivery at WhatsApp scale | Fast webhook ack + background processing, retry/backoff, message splitting |
| Cloud-portable deployment | Docker + Cloud Run, or local scripts with ngrok tunnel |
